### 2. Анализ уязвимостей (Exploit-DB)
На основе полученных версий ПО был проведен анализ базы данных [Exploit-DB](https://exploit-db.com). Выявлены следующие критические уязвимости:

1. **vsftpd 2.3.4 - Backdoor Command Execution**
   * **Описание:** В исходный код данной версии FTP-сервера был внедрен бэкдор. При отправке имени пользователя, содержащего символы `:)`, активируется скрытый шелл на порту 6200 с правами root.
   * **Ссылка на exploit-db:** [EDB-ID 49757](https://exploit-db.comexploits/49757)
2. **UnrealIRCd 3.2.8.1 - Backdoor Command Execution**
   * **Описание:** Официальный архив версии IRC-сервера содержал вредоносную модификацию. Отправка специальной строки («AB») позволяет выполнять произвольные системные команды.
   * **Ссылка на exploit-db:** [EDB-ID 16922](https://exploit-db.comexploits/16922)
3. **Java RMI Server - Remote Code Execution**
   * **Описание:** Служба по умолчанию доверяет удаленным запросам на загрузку классов, что позволяет злоумышленнику выполнить произвольный Java-код в контексте запущенного процесса.
   * **Ссылка на exploit-db:** [EDB-ID 48503](https://exploit-db.comexploits/48503)

## Задание 2. Сравнение режимов сканирования Nmap в Wireshark

Для анализа сетевого трафика были запущены сканирования в режимах **SYN**, **FIN**, **Xmas** и **UDP**. Результаты перехвата пакетов в Wireshark показали существенные различия в логике взаимодействия.

### Сводная таблица поведения сетевого трафика

| Режим сканирования | Команда Nmap | Флаги в запросе | Ответ (Порт Открыт) | Ответ (Порт Закрыт) |
| :--- | :--- | :--- | :--- | :--- |
| **SYN** | `nmap -sS` | `SYN` | `SYN-ACK` $\rightarrow$ Сканер шлет `RST` | `RST` |
| **FIN** | `nmap -sF` | `FIN` | **Ответ отсутствует** (Игнорируется) | `RST` |
| **Xmas** | `nmap -sX` | `FIN, PSH, URG` | **Ответ отсутствует** (Игнорируется) | `RST` |
| **UDP** | `nmap -sU` | *Нет (UDP-пакет)* | **Ответ отсутствует** или ответ службы | `ICMP Destination Unreachable` |

### Детальный разбор механизмов


* **SYN (Полуоткрытое):** Наиболее популярный метод. Сканер отправляет `SYN`, получает `SYN-ACK` (подтверждающий, что порт открыт), но вместо завершения соединения отправляет `RST`. Полноценная сессия не создается, что снижает вероятность фиксации в логах приложений.
* **FIN:** Сканер отправляет пакет «закрытия» соединения. По правилам RFC, открытый порт Linux-системы должен сбросить такой пакет без ответа, а закрытый — вернуть `RST`.
* **Xmas:** Назван так, потому что пакет «мигает» флагами `FIN`, `PSH` и `URG`, как новогодняя елка. Механизм определения аналогичен FIN-сканированию: открытый порт молчит, закрытый — возвращает `RST`.
* **UDP:** Так как протокол работает без установления соединения, сканер отправляет пустой датаграмм. Если порт закрыт, операционная система возвращает ошибку `ICMP (Type 3, Code 3)`. Если ответа нет, Nmap интерпретирует это как `open|filtered`.

Ответы от сервера есть всегда, если порт закрыт (сервер всегда явно говорит «нет» с помощью пакетов RST или ICMP).Но если порт открыт, сервер отвечает только на SYN-сканирование (SYN-ACK). На сканирования FIN и Xmas открытый порт не отвечает ничего, и именно по этой тишине Nmap понимает, что порт доступен.
